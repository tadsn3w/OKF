# Standards

Reference documents that define conventions, formats, and protocols this
project follows.  Dropping a spec here gives the AI direct access to the
canonical rules — no guessing, no web lookups.

## What goes here

- Downloaded specs (Conventional Commits, SemVer, Conventional Comments, etc.)
- Team-internal style guides
- Format contracts (commit message templates, changelog formats, ADR format)
- Protocol references (RFC summaries, API style guides)

## How to add

Download the spec as Markdown (or convert it) and drop it here.  Name the file
descriptively: `conventional-commits.md`, `semver.md`, `adr-format.md`.

## How the AI uses this

When a task involves committing, versioning, changelogs, or any convention
governed by one of these standards, the AI reads the relevant file here
before acting.
